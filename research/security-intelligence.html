Security Intelligence

Applying the NLP and large language model techniques developed in HPC research to a new domain: cybersecurity intelligence. Automating the analysis and clustering of threat documents to extract actionable intelligence from large, unstructured data sources — at machine speed and scale.

📖 Overview

The cybersecurity domain faces an information overload problem: threat intelligence analysts must process enormous volumes of reports, advisories, vulnerability disclosures, and incident write-ups — far more than any human analyst can read and synthesize in a timely manner. The challenge is not a shortage of information, but a shortage of automated tools to extract signal from noise and organize it into actionable intelligence.

Leo's security intelligence research transfers LLM and NLP expertise accumulated in the HPC domain to address this challenge. The core technique is LLM-driven document clustering: using large language model embeddings to represent threat documents in a semantically rich vector space, then applying clustering algorithms to automatically group related documents — malware families, attack campaigns, vulnerability types — without requiring manually crafted taxonomies or labeled training data. This unsupervised approach scales to large corpora and adapts automatically to emerging threats.

The key insight driving this work is that the NLP and LLM infrastructure developed for HPC code analysis — understanding technical documents, extracting key concepts, grouping related artifacts — transfers naturally to the security domain. Both domains deal with highly technical, domain-specific text where general-purpose NLP models struggle, and both benefit from the same approach: fine-tuning or prompt-engineering LLMs with domain context, then applying clustering or classification to organize the results. Lawrence Livermore National Laboratory's unique position at the intersection of HPC and national security makes this cross-domain research a natural fit.

🔬 Technical Approach

The LLM-driven clustering pipeline processes security documents through several stages. First, documents are embedded using a large language model (or a security-domain fine-tuned variant) to produce dense semantic vector representations that capture meaning beyond keyword matching. These embeddings encode relationships between technical concepts — two documents about the same exploit technique will be close in embedding space even if they use different terminology.

Clustering algorithms (k-means, HDBSCAN, or hierarchical methods) then group the embedded documents, with cluster quality evaluated using both quantitative metrics (silhouette score, clustering stability) and human expert assessment. The resulting clusters can be labeled, summarized, and linked to known threat intelligence frameworks (e.g., MITRE ATT&CK), providing a structured, searchable intelligence picture from raw, unstructured text corpora.

A distinctive aspect of this work is the cross-domain transfer story: insights from applying LLMs to HPC code (handling highly technical, domain-specific text; dealing with specialized vocabulary; clustering related artifacts by semantic similarity) transfer directly to the security domain. This cross-pollination between HPC AI research and security informatics is a hallmark of LLNL's multidisciplinary research environment.

💡 Impact & Insights

The same LLM capabilities that help HPC developers understand compiler reports can help security analysts understand threat landscapes — the technical depth required is similar, only the domain changes.
  • Cross-domain transfer of LLM techniques demonstrates that investments in AI for HPC have broader payoffs — the infrastructure, prompting strategies, and evaluation methods developed for code analysis apply directly to security text analysis.
  • Unsupervised clustering is strategically important for security intelligence because labeling training data is expensive, time-consuming, and falls behind the pace of emerging threats — the system works without requiring labeled examples.
  • LLM embeddings capture semantic meaning that keyword-based approaches miss: two documents about the same threat actor may use entirely different terminology, yet LLM embeddings place them close in vector space.
  • This work reflects LLNL's dual mission in both scientific computing and national security — applying computational advances from one domain to strengthen capabilities in the other.
  • The ISI 2025 publication introduces this research to the security informatics community, opening doors for collaboration between the HPC AI and cybersecurity research communities.
